## Generated Documentation

These files are created by the system during operation:

### traces/
Directory containing per-user decision trace JSONs:
- `traces/{user_id}.json` - Complete audit trail of persona assignment and recommendation generation

### eval_summary.md
Generated by evaluation harness (`eval/run.py`) with:
- Coverage, explainability, latency, fairness metrics
- Demographic distribution analysis
- System performance summary

### test_results.md
Generated by test suite (`pytest`) with:
- Test pass/fail status
- Coverage reports
- Execution times

### fairness_report.md
Generated by evaluation harness with:
- Demographic parity analysis
- Persona distribution by group
- Flagged disparities

## Documentation Standards

### Format
- Use Markdown (.md) for all documentation
- Include last updated date in headers
- Use tables for structured data
- Include code examples where helpful

### Naming
- `snake_case` for generated files
- `Title Case` for manual documentation
- Prefix traces with user_id: `user_0001.json`

### Updates
- Update decision_log.md when making design choices
- Regenerate eval_summary.md after each evaluation run
- Update limitations.md as scope changes
